# Ideas

Reviewed ideas and sparks worth keeping even when they are not immediately
actionable.

## Memory Units Instead Of Session-Sized Skills

Source: VibeWiki design conversation, 2026-06-19.

A useful VibeWiki memory unit should be as small as the reusable insight allows.
Some session content should become skilllets or prompt patterns, but much of it
should remain as knowledge, issues, todos, ideas, research notes, or directions.

This helps avoid two failure modes:

- creating overly specific skills that are rarely reused
- losing small but important sparks because they were not executable enough to
  become a skill

## Human And Agent Entrances

VibeWiki should keep two reuse entrances:

- a human-facing entrance that answers natural-language questions from reviewed
  and clearly marked candidate memory
- an agent-facing entrance that returns compact JSON/YAML context packs so AI
  agents can search relevant skills and project knowledge without long manual
  prompts
